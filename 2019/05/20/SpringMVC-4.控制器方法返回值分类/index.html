SpringMVC 4.控制器方法返回值分类

发表于 2019-05-20 | 更新于 2019-05-21 | 分类于 SpringMVC | 评论数：

字符串

void

Servlet 原始 API 可以作为控制器中方法的参数
但此方法不会调用视图解析器

ModelAndView

ModelAndView 是 SpringMVC 为我们提供的一个对象，该对象也可以用作控制器方法的返回值
三个常用方法：

  • modelAndView.addObject() 通过调用ModelMap.addAttribute 向Request域中添加键值对
  • modelAndView.getModelMap()获取ModelMap对象
  • modelAndView.setViewName()设置视图声明跳转文件名 否则默认为 请求路径与RequestMapping的值保持一致xxx.jsp

ResponseBody 响应 json 数据

该注解用于将 Controller 的方法返回的对象，通过 HttpMessageConverter 接口转换为指定格式的
数据如：json,xml 等，通过 Response 响应给客户端
json字符串和JavaBean对象互相转换的过程中，需要使用jackson的jar包

DispatcherServlet会拦截到所有的资源，导致一个问题就是静态资源（img、css、js）也会被拦截到，从而
不能被使用。解决问题就是需要配置静态资源不进行拦截，在springmvc.xml配置文件添加如下配置

mvc:resources标签配置不过滤

  1. location元素表示webapp目录下的包下的所有文件
  2. mapping元素表示以/static开头的所有请求路径，如/static/a 或者/static/a/b
